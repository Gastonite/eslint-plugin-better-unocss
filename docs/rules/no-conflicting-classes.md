# better-unocss/no-conflicting-classes

Disallow classes that generate the same CSS property.

## Rule Details

This rule detects classes that conflict by setting the same CSS property. When multiple classes set the same property, only the last one takes effect (CSS cascade), so earlier ones are unnecessary.

### Examples

#### Invalid

```tsx
// Both set 'display'
cn`flex block`

// Both set 'color'
cn`text-red-500 text-blue-500`

// Both set 'padding'
cn`p-4 p-8`

// Both set 'justify-content'
cn`justify-start justify-center`
```

#### Valid

```tsx
// Different properties
cn`flex text-red-500 p-4`

// Same property but different variants (different states)
cn`flex hover:block`

// Same property but different breakpoints
cn`md:flex lg:block`
```

## Options

```typescript
type NoConflictingClassesOptions = {
  selectors?: Selector[]
}
```

### `selectors`

Custom selectors to detect class strings. See [Selectors](../selectors.md) for details.

## Variant Handling

Two classes only conflict if they have the **same variant chain**:

| Classes | Conflict? | Reason |
|---------|-----------|--------|
| `flex block` | Yes | Same state (no variant) |
| `flex hover:block` | No | Different states |
| `hover:flex hover:block` | Yes | Same variant |
| `md:flex lg:block` | No | Different breakpoints |
| `dark:flex dark:block` | Yes | Same variant |
| `sm:hover:flex sm:hover:block` | Yes | Same variant chain |

## Autofix

This rule automatically removes conflicting classes, keeping the **last occurrence** (CSS cascade behavior):

```tsx
// Before
cn`text-red-500 text-blue-500`

// After (autofix) - keeps last
cn`text-blue-500`
```

```tsx
// Before
cn`p-4 flex p-8 items-center`

// After (autofix)
cn`flex p-8 items-center`
```

## How It Works

1. Parses each class with UnoCSS engine
2. Extracts the CSS properties generated by each class
3. Groups classes by `variant:property`
4. Reports conflicts when multiple classes generate the same property in the same variant context

## Error Message

```
Conflicting classes for display: flex, block
Conflicting classes for color, --un-text-opacity: text-red-500, text-blue-500
```

## When Not To Use It

- If you intentionally override classes for specificity
- If you use utility classes that are meant to override each other
